# 第5课：语言模型：预测下一个词

## 内容引入

**3 分钟互动：你来接下一句**

教师说出几个开头，让学生快速接词：

```text
床前明月……
今天天气很好，我们一起去……
我今天早上喝了一杯……
```

学生会发现，有些开头几乎只有一个高频答案，有些开头有多个合理答案。引导问题：大语言模型写文章，会不会也是在不断预测后面最可能出现什么？

## 知识点讲授

### 1. 人类也会根据上下文预测

当我们听到“床前明月”，很容易接出“光”。当我们听到“今天天气很好，我们一起去”，可能会接“操场”“散步”“打球”“公园”。这些答案来自我们对语言、场景和经验的判断。

预测下一个词并不只是背诵，它依赖上下文。

### 2. 语言模型做的事情

语言模型的核心任务可以理解为：

> 看前面的文字，猜下一个最可能出现的词。

例如：

```text
我今天早上喝了一杯……
```

模型会根据训练中见过的大量语言规律判断：

- “牛奶”的可能性较高；
- “咖啡”的可能性也较高；
- “石头”“铅笔”的可能性很低。

模型并不是只知道一个答案，而是会对很多候选词估计可能性。

### 3. 为什么预测下一个词能写文章

语言模型不是只预测一次，而是连续预测。每生成一个词，这个词又会成为新的上下文，继续影响后面的预测：

```text
人工智能 → 正在 → 改变 → 人类 → 的 → 生活
```

一次次预测连接起来，就形成句子、段落甚至文章。问答、写作、总结和翻译，都可以看作在特定要求下连续生成合适文本。

### 4. 它是不是在真正思考

大模型生成语言看起来像思考，但底层机制更接近：

> 根据上下文和训练中见过的大量语言规律，生成最可能合理的后续内容。

这不等于它像人一样有真实经历、情感和意识。它可能写得很流畅，也可能因为预测错误或事实核查不足而给出错误答案。

### 5. 概率很重要

同一个开头可能有多个合理后续。例如“我喜欢吃”后面可以是“苹果”“面条”“火锅”。语言模型会根据上下文给不同词分配不同概率。生成文本时，模型会在高概率答案中选择合适内容。

## 互动强化

**20 分钟活动：全班做人肉语言模型**

### 环境一：每个学生都可以使用自己的电子设备

1. 教师在共享文档中给出 5 个句子开头。
2. 学生各自提交自己认为最合理的下一个词。
3. 教师汇总词频，展示每个候选词的“概率”。
4. 小组讨论：为什么有些词票数高？如果增加更多上下文，答案会不会更集中？
5. 教师逐步延长句子，让学生连续预测 5 轮，观察句子如何生成。
6. 总结：语言模型通过连续预测生成文本，预测依赖上下文。

### 环境二：只有老师可以使用电子设备

1. 教师投影句子开头。
2. 学生举手投票选择下一个词，教师在屏幕或黑板记录票数。
3. 教师选择票数最高的词加入句子，进入下一轮预测。
4. 连续 5 轮后，全班读出生成句子。
5. 教师展示另一个上下文更明确的开头，让学生比较预测分布是否更集中。

### 环境三：不能使用电子设备

1. 教师在黑板写出句子开头。
2. 每组把下一个词写在纸条上交给教师。
3. 教师按答案分类计数，在黑板画出简单条形统计。
4. 选择最高票词继续生成，重复 4-5 轮。
5. 教师总结“高票词”就是更可能的候选，连续选择会生成完整文本。

## 课后提升

**20 分钟小作业：做一次小型文本生成实验**

写出 3 个句子开头，每个开头邀请 3 位同学或家人补充下一个词。记录不同人的答案，并回答：

1. 哪个开头的答案最集中？
2. 哪个开头的答案最分散？
3. 上下文越清楚，预测是否越容易？

有能力的同学可以查阅“n-gram 语言模型”，了解早期语言模型如何根据前几个词预测下一个词。
