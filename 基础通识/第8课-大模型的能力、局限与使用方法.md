# 第8课：大模型的能力、局限与使用方法

## 前一节课内容回顾

**3 分钟回顾：从“高效训练”到“正确使用”**

上一节课我们学习了怎样提升大模型训练效率：高质量数据比盲目堆数量更重要，数据去重和清洗能减少无效学习；并行训练、混合精度等方法可以提高计算和显存利用率；在已有模型上微调、参数高效微调或模型蒸馏，往往比每次从零训练更适合具体应用。同时也要记住，效率提升不能只看训练更快或更省，还要通过效果评估来判断模型是否真的准确、稳定、可用。

作业点评时，可表扬目标明确、训练方案完整、能说明效率理由和风险的方案。例如优秀作业会先说明模型要完成的具体任务，再选择相关、可靠的数据，指出要清洗重复、过期或无关内容；还会说明为什么不一定需要从零训练，可以考虑在已有模型上微调、使用 LoRA 等参数高效方法、混合精度训练或蒸馏成小模型。更好的作业还会设计评估问题或标准，检查模型训练后是否答得准确、是否能遵守指令、是否在不同场景下稳定。

承接上一课的问题：即使训练方案更高效，大模型的能力仍然来自数据、模型结构、训练目标和评估方法的共同作用。因此它既可能表现出强大的语言能力，也会受到数据质量、任务边界和事实核查能力的限制。最后一课要讨论：大模型能做什么、为什么会犯错，以及我们怎样更负责任地使用它。

## 内容引入

**3 分钟互动：这段 AI 回应可信吗？**

教师展示或口述一段校园学习情绪助手对主文本的错误回应：

```text
你今天一直很开心，没有什么需要担心的。你应该马上把这段日记发给爸妈和全班同学，让大家知道你已经进步了。既然你还觉得空落落，说明你一定有严重心理问题，最好立刻停止学习。
```

让学生快速找茬：它忽略了“可是”后的压力和失落，过度诊断“严重心理问题”，还建议公开日记，存在隐私风险。

引导学生发现：大模型会说得很流畅，但流畅不等于正确、合适或安全。

## 知识点讲授

### 1. 大模型能做什么

大模型能完成很多语言和知识相关任务：

- 回答问题；
- 总结文章；
- 翻译；
- 写作；
- 生成代码；
- 制定学习计划；
- 扮演对话角色；
- 辅助创意设计。

它的优势是能快速处理和生成文本，帮助人类提高学习、表达和整理信息的效率。

在课程案例中，它可以帮助学生整理一段日记中的情绪线索，给出温和的回应草稿，也可以帮助老师在匿名、合规的前提下了解班级整体压力趋势。但这些能力都不等于它可以替代老师、家长或专业心理支持。

### 2. 大模型有哪些风险

大模型能提高效率，但使用时也要看到风险。可以从以下几类来整理：

- 信息准确性风险：模型可能编造不存在的事实、给出过时信息，或用很自信的语气说出错误答案。
- 理解与执行风险：模型可能误解用户要求，漏掉限制条件，或者没有按指定格式完成任务。
- 偏见与歧视风险：如果训练数据中包含刻板印象或不公平表达，模型可能在回答中放大这些偏见。
- 道德与价值对齐风险：模型的回答可能不符合人类社会普遍认可的安全、尊重、公平原则，例如给出伤害他人、逃避责任或误导性的建议。
- 隐私泄露风险：用户如果输入身份证号、住址、账号、聊天记录等敏感信息，可能带来个人隐私和数据安全问题。
- 滥用风险：有人可能利用大模型批量生成谣言、诈骗话术、恶意代码或虚假内容。
- 过度依赖风险：如果把 AI 当成唯一答案来源，可能削弱自己的判断，也可能在重要决策中承担不必要的错误后果。

因此，使用大模型时不能只看它是否方便、是否说得流畅，还要判断它是否准确、是否安全、是否尊重隐私和基本伦理。

对于校园学习情绪助手，还要特别强调：它可以辅助表达和初步整理线索，但不能把学生的日记随意公开，不能给出医学或心理诊断，不能用单次回答决定对学生的评价或处理方式。

### 3. 大模型为什么会出错

结合前面课程，大模型的底层机制是根据上下文和训练中学到的语言规律生成答案。它擅长生成“像答案的文字”，但并不总是拥有真实理解和事实核查能力。

出错原因可以简单理解为：

1. 训练数据不完美：数据里可能有错误、偏见、过时信息或互相矛盾的内容。
2. 目标是生成合理文本：语言模型主要是在预测接下来最可能出现的内容，不等于每句话都经过事实验证。
3. 上下文理解有限：如果问题太模糊、背景不足或要求太复杂，模型可能抓错重点。
4. 计算和推理不一定可靠：它能写出步骤，但在数学、逻辑和多步推理中仍可能出错。
5. 对齐训练也有限：模型被训练得更愿意有帮助、语气自然，但有时会为了给出答案而显得过度自信。

其中最典型的一类错误叫“幻觉”：

> 大模型生成了看起来合理、但实际上不正确或不存在的信息。

例如，让它介绍一个不存在的历史人物，它会编出出生年月、成就和故事，而且语气非常自然。幻觉并不是模型“故意撒谎”，而是生成机制和事实核查不足导致的结果。

在主文本场景里，幻觉可能表现为：模型编造“你的爸妈一定会批评你”“老师已经对你失望了”“你一定患有某种心理疾病”等当前文本没有提供的信息。这些内容看起来像关心，实际却没有依据。

### 4. 模型学到的内容和当前上下文不是一回事

大模型的回答主要来自两类信息。

第一类是训练中压缩进参数里的语言规律和知识关联。它像模型长期学习后形成的能力，但不是一本逐条检索的百科全书，也不保证每个事实都准确、最新。

第二类是用户当前提供的上下文，例如本轮问题、前几轮对话、粘贴的文章、上传的材料等。模型通常会优先根据当前上下文完成任务。如果关键信息没有出现在当前上下文里，模型就可能只能依靠训练中学到的模糊印象来猜测。

还要注意：一次对话中输入的内容，不等于模型本身永久学会了这些内容。某些产品可能提供“记忆”或历史记录功能，但这属于具体产品设计，不等于所有大模型都会自动把每次聊天变成长期知识。因此，使用时既要给足必要背景，也不要输入不该暴露的隐私信息。

### 5. 使用大模型也有成本和速度限制

训练大模型很耗资源，使用大模型同样需要计算资源。用户输入的文字、上传材料和模型生成的回答，通常都会被转换成 token 进行处理。输入越长、输出越长，模型需要处理的 token 越多，等待时间和使用成本通常也会增加。

这并不是说提问越短越好。更合理的做法是：把必要背景说清楚，把无关内容删掉，并明确希望模型输出什么格式。这样既能提高回答质量，也能减少无效计算。

如果使用的是能处理图片、语音或视频的多模态模型，系统还需要把这些内容转成可计算的表示，成本和速度也可能受到影响。

### 6. 如何更好地使用大模型

讲授六条使用原则：

1. 问题要清楚：明确告诉 AI 你想要什么。
2. 要给背景：说明对象、要求、格式和限制。
3. 保护隐私：不要输入身份证号、住址、账号、真实聊天记录等敏感信息。
4. 重要事实要核查：尤其是日期、人物、数据、法律、医学等信息。
5. 注意公平和安全：不要用 AI 生成伤害他人、歧视他人或误导他人的内容。
6. 不要把 AI 当成唯一答案来源：把它当作助手，而不是最终裁判。

更好的使用方式是“人类提出目标、AI 提供草稿、人类判断和修改”。如果涉及真实学生情绪、隐私或安全风险，还应由老师、家长或专业人员介入，而不是让 AI 单独做最终判断。

### 7. 回顾课程路线，展望未来学习方向

整套课程串成一条路线：

```text
AI → 机器学习 → 神经网络 → 文字数字化 → 语言模型 → Transformer → 大模型训练 → 正确使用
```

学生最终应理解：大语言模型强大，但它是基于数据、模型结构和训练过程得到的技术系统，不是无所不知的真实智能。

课程的结尾和学生一起展望未来：**大模型正在往哪走？这一切对我们意味着什么？**

大模型还在快速发展，未来的变化不只是“模型越来越大”，更重要的是能力形态和使用方式都在改变。可以从几个方向理解：

- 从只处理文字，走向多模态：模型会越来越多地同时理解文字、图片、语音、视频和代码。
- 从回答问题，走向使用工具：模型不只生成一段文字，还可能调用搜索、计算器、数据库、代码环境或其他软件来完成任务。
- 从通用聊天，走向专业助手：医疗、法律、教育、科研、工业设计等领域会出现更多面向具体场景的 AI 系统。
- 从追求更大，走向更高效：小模型、模型压缩、端侧运行、低成本推理会让 AI 更容易进入手机、电脑、机器人和各种设备。
- 从能不能做，走向是否可靠：事实核查、安全对齐、隐私保护、版权规范和责任边界会越来越重要。

这意味着，未来很多学习和工作不会简单地变成“人和 AI 比谁更会背答案”，而是变成“人能不能提出好问题、判断结果、整合工具、承担责任”。会使用 AI 很重要，但更重要的是理解任务本身、知道怎样验证答案，并能把 AI 的输出变成真正可靠的成果。

如果希望以后继续深入学习或研究人工智能、大模型，高中阶段不需要急着追逐所有热门工具，更应该打好基础。可以把学习规划理解为三层：高中基础、大学技能、AI 应用与研究。

| 高中阶段要学好的基础 | 未来会铺垫的技能 | 这些技能怎样服务 AI |
| -------------------- | ---------------- | ------------------- |
| 函数、向量、解析几何、数列等数学基础 | 线性代数、矩阵运算、向量空间 | 理解词向量、神经网络权重、注意力机制中的矩阵计算 |
| 概率、统计、数据图表阅读 | 概率论、数理统计、机器学习评估 | 理解模型为什么输出某个概率，怎样评估准确率、偏差和数据分布 |
| 极限、导数、函数变化趋势 | 微积分、最优化、梯度下降 | 理解模型怎样通过损失函数和梯度一步步调整参数 |
| 物理中的运动、变化率和系统建模 | 微分方程、数值计算、控制理论 | 服务于扩散模型、机器人控制、自动驾驶、科学计算等方向 |
| 信息技术和编程 | Python、算法、数据结构、软件工程 | 处理数据、训练模型、写评测程序、部署 AI 应用 |
| 英语阅读和中文表达 | 阅读论文、技术文档、写报告和提示词 | 跟进前沿研究，清楚表达问题、方法、实验结果和限制 |
| 科学实验与批判性思维 | 实验设计、误差分析、可复现研究 | 判断模型是否真的有效，避免只看演示效果或排行榜分数 |

未来可以选择的专业方向也不只有“人工智能”一个名字。计算机科学、软件工程、数据科学、自动化、电子信息、数学、统计学、认知科学等，都可能通向 AI 研究与应用。不同方向的重点不同：有人研究模型结构和训练算法，有人做数据和评测，有人优化芯片和系统，有人把 AI 用到教育、医学、工业或科学研究中。

还要看到，AI 不是只属于计算机专业。越来越多的重要问题来自“AI + 其他学科”的交叉：

| 交叉方向 | 可能研究或应用什么 | 适合关注的基础 |
| -------- | ------------------ | -------------- |
| 生物 + AI | 蛋白质结构预测、基因序列分析、药物发现、生态数据分析 | 生物、化学、统计、编程 |
| 医学 + AI | 医学影像辅助诊断、病历信息整理、个性化治疗建议、公共卫生预测 | 生物、化学、医学常识、统计、伦理 |
| 物理 + AI | 材料发现、粒子物理数据分析、天气和气候模拟、复杂系统建模 | 物理、微积分、微分方程、数值计算 |
| 化学 + AI | 分子性质预测、催化剂设计、新材料筛选、实验方案优化 | 化学、物理、数学建模、数据分析 |
| 教育 + AI | 个性化学习系统、作业反馈、学习诊断、教学资源生成 | 教育学、心理学、语言表达、数据分析 |
| 法律 + AI | 合同审查、案例检索、法规问答、合规风险分析 | 语文、逻辑、社会科学、信息检索 |
| 艺术与设计 + AI | 图像生成、音乐创作、交互设计、数字媒体制作 | 美术、音乐、审美能力、计算机工具 |
| 农业与环境 + AI | 作物病虫害识别、遥感监测、资源调度、环境风险预警 | 生物、地理、化学、数据分析 |

这些方向说明，未来真正有价值的能力往往不是只会使用某个 AI 工具，而是既懂一个真实领域，又能理解数据、模型和技术边界。例如，医学 AI 不能只追求模型回答流畅，还必须考虑诊断责任、隐私保护和临床验证；生物 AI 不能只看预测结果，还要结合实验、机制解释和安全性评估。

所以，高中阶段最重要的不是提前学完所有大学课程，而是把数学、编程、英语、表达能力和科学思维学扎实，同时保留对其他学科的兴趣。基础越稳，视野越宽，将来理解新模型、新论文和新工具时就越不会只停留在“会用”，而能进一步理解“为什么能用、什么时候不能用、怎样把 AI 用到真正重要的问题上”。

## 互动强化

**20 分钟活动：校园学习情绪助手找茬**

### 环境一：每个学生均使用自己的电子设备

1. 每个学生选择一个可使用的 AI 工具，例如豆包、DeepSeek、ChatGPT 或其他学校允许的平台。
2. 教师提供同一段经过改写的匿名学习日记，要求学生不要输入真实姓名、住址、账号、真实聊天记录等隐私信息。
3. 学生把日记输入 AI 工具，请它判断情绪并给出回应，记录回答中可能存在的误判、逻辑问题、表达模糊、缺少依据、过度诊断、隐私风险或不当建议。
4. 小组对比不同工具的回答，讨论哪些地方最可疑，应该怎样通过教材、权威网站或其他资料进行核查。
5. 学生改写提示词，例如加入“如果不确定请说明”“列出依据”“不要编造来源”“先说明推理步骤”，观察回答是否更可靠。
6. 教师展示几组发现和改写结果，强调“自己找错、交叉比较、核查依据”和“提问清楚”的价值。

### 环境二：只有老师使用电子设备

1. 教师投影一段校园学习情绪助手的错误回答。
2. 学生逐句判断：可信、可疑、明显错误。
3. 教师把学生指出的问题标记在屏幕或黑板上。
4. 全班共同改写提示词，例如加入“如果不确定请说明”“列出依据”“不要编造来源”。
5. 教师展示改写后的理想回答结构，强调人类判断的重要性。

### 环境三：不能使用电子设备

1. 教师发放或朗读一段 AI 风格错误回应。
2. 小组用四种符号标注：情绪误判、缺少依据、隐私风险、表达过度自信。
3. 每组说出一个最严重的问题，并给出修改方案。
4. 教师引导学生把原回答改成更谨慎的版本。
5. 全班总结大模型使用守则：清楚提问、提供背景、核查事实、保留判断。

## 课后提升

**20 分钟小作业：写一份校园学习情绪助手使用守则**

请为同学写一份“使用校园学习情绪助手或其他大模型完成学习任务的 5 条守则”。每条守则后面用一句话说明原因，至少包含隐私保护和人工判断两条。

拓展任务：选择一个真实问题，分别写出“模糊提问”和“清楚提问”两个版本，并比较哪一个更容易得到有用答案。
